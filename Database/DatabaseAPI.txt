# 项目重构需求



📘 项目名称：基于支付宝小程序云开发框架的电商平台Demo
🧾 一、项目概述
🌟 项目目标：
开发一个基于 支付宝小程序云开发框架 的电商平台Demo，使用 支付宝开放平台提供的云开发能力（包括云函数、云数据库、云存储等），构建一个具备商品分类、展示、购物车、下单、支付、订单管理等功能的完整闭环电商流程。

🧩 二、功能模块与评分标准
下方为文档中明确规定的评分维度、功能要求及实现标准。

📌 1. 首页功能（总分 40分）
✅ 1.1 分类边栏（左侧导航）
使用 getCategory 云函数从数据库 t_category 表获取所有分类。

分类以垂直列表形式展示，推荐使用 <scroll-view> 组件保证长列表可滑动。

点击分类项后，右侧九宫格内容更新为该分类下的子类。

✅ 1.2 九宫格内容区（右侧）
展示选中分类下的所有子类，调用 getSubCategory(categoryId) 云函数，从 t_sub_category 表获取数据。

每个子类显示一个图片和文字组成九宫格布局。

图片使用云存储中公有链接，确保能在小程序中显示。

点击任意子类，跳转至商品列表页 (list) 并携带该子类信息作为参数。

✅ 1.3 轮播图展示
顶部使用 <swiper> 组件展示三张图片。

第一张必须是开发者本人照片（上传到云存储，公有访问）。

其余两张可以是电商宣传图或主题相关图，所有图片均需使用云存储公有链接。

📌 2. 商品列表页（总分 20分）
✅ 2.1 数据获取与展示
页面通过 getProduct(subCategoryId) 云函数获取商品列表数据（来源于 t_product 表）。

商品列表为组件化展示，每个商品显示：

商品图（云存储URL）

名称

价格

✅ 2.2 输入框展示
页面顶部有一个 input 输入框，默认提示词为所选子类名称。

仅展示用途，无需实现搜索功能。

✅ 2.3 点击进入详情页
点击任意商品项跳转至 detail 页面，并将商品ID传入。

✅ 2.4 分页加载
实现“上拉加载更多”的逻辑，即分页查询商品（limit + offset）。

数据可以重复使用，用于演示分页效果。

📌 3. 商品详情页（总分 10分）
✅ 3.1 页面内容
页面展示内容包括：

商品标题

商品图片轮播

商品详情介绍

页面底部显示一个自定义组件 detail-settle（商品操作栏）。

✅ 3.2 加入购物车功能
detail-settle 组件包含“加入购物车”按钮。

点击后调用 addToCart(productId) 云函数，添加记录到 t_cart 表。

可在 t_cart 中增加字段：数量、是否选中。

📌 4. 购物车页（总分 40分）
✅ 4.1 数据获取
页面通过 getCartList() 云函数获取购物车内容（t_cart 表 + t_product 联合查询）。

每项展示为复用商品Item组件，需包括：

商品图、名、价格

数量调节（+ / -）

是否选中（checkbox）

✅ 4.2 底部结算栏（组件 cart-settle）
自定义组件显示：

合计金额（计算已选中商品价格总和）

“结算”按钮

勾选商品或数量变化时，金额自动更新。

✅ 4.3 删除操作
提供删除功能（如滑动删除按钮）。

删除后页面应自动刷新。

📌 5. 订单与支付模块（总分 40分）
✅ 5.1 跳转订单页
点击“结算”按钮后，跳转至 order 页面。

页面显示订单详情卡片（商品名汇总/金额）。

✅ 5.2 创建订单
点击“立即支付”按钮：

调用 createOrder(title, amount) 创建订单，记录至 t_order 表。

调用 myPay() 模拟支付，返回模拟订单编号（如 out_trade_no）。

✅ 5.3 更新订单状态
支付成功后调用 updateOrder(orderId) 云函数，将订单状态改为“待发货”。

📌 6. 我的页面（总分 10分）
✅ 6.1 订单状态统计
页面调用 getOrder() 云函数，获取用户所有订单数据（t_order 表）。

按照不同订单状态分类展示（如：待付款、待发货、已发货）。

每个分类项使用 Antd-mini 的 Badge 组件展示该状态下订单数量。

✅ 6.2 状态变化响应
若手动在数据库中更改某订单状态，前端应能正确显示数量变化。

📌 7. TabBar 和导航栏要求
✅ 7.1 TabBar 配置
至少包含3个Tab项：首页、购物车、我的。

每项包含图标，建议使用本地图片或云存储URL。

在 app.json 中配置 tabBar。

✅ 7.2 导航栏标题
所有页面的顶部导航栏文字必须设置为你自己的长学号。

设置方式：每页的 navigationBarTitleText 属性。

🧱 三、数据库结构说明
表名	字段	说明
t_category	_id, name	商品一级分类
t_sub_category	_id, category_id, name, url	子分类，图片地址为URL
t_product	_id, sub_category_id, name, price, url	商品信息
t_cart	_id, product_id, quantity, isChecked	用户购物车内容
t_order	_id, title, amount, status	订单记录，status为订单状态

☁️ 四、云函数说明
云函数	参数	功能	数据表
getCategory	无	获取一级分类	t_category
getSubCategory	categoryId	获取子分类列表	t_sub_category
getProduct	subCategoryId	获取商品	t_product
addToCart	productId	添加购物车项	t_cart
getCartList	无	获取购物车内容	t_cart + t_product
createOrder	title, amount	创建订单	t_order
myPay	orderId	模拟支付流程	无
updateOrder	orderId	修改订单状态	t_order
getOrder	无	获取订单列表	t_order
